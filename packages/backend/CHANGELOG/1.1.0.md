# 1.1.0 (2025-12-01)

## Added

### Spec-First Branching Workflow

- **새로운 워크플로우**: dev 브랜치에서 Spec 작성 → 원격 푸시 → Feature 브랜치 생성
- **목적**: 다른 작업자도 특정 도메인의 Spec을 공유받을 수 있도록 함
- Feature 브랜치에서는 실제 코드 구현만 수행

## Changed

### git-workflow/references/branch-strategy.md - Spec-First Branching Rule 추가

- 워크플로우 다이어그램 (ASCII art)
- 브랜치별 허용/금지 작업 테이블
- dev: Spec 작성, 설정 변경
- feature/*: 코드 구현, 테스트

### _shared/commit-guide.md - Spec 커밋 규칙 추가

- Spec 커밋 메시지 형식: `📝 #{이슈번호} Add spec for {도메인}`
- dev 브랜치에서 이슈 번호 추출 방법
- Spec 커밋 워크플로우 가이드

### CLAUDE.md - 워크플로우 다이어그램 업데이트

- Spec-First Branching 다이어그램 추가
- 브랜치별 작업 구분 테이블 추가
- SDD는 dev 브랜치, ADD는 feature 브랜치 명시
