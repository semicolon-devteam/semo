# v0.22.0

**Release Date**: 2025-12-09

## Added

### Cross-Package Routing Guide

다른 패키지의 전문 영역 요청 시 인계 및 권유 가이드 추가:

#### 파일 추가

| 파일 | 설명 |
|------|------|
| `_shared/cross-package-routing.md` | 크로스 패키지 인계 가이드라인 |

#### 주요 기능

- **전문 영역 매트릭스**: 각 패키지별 전문 영역 및 키워드 정의
- **인계 트리거 키워드**: 다른 패키지로 인계해야 하는 키워드 목록
- **인계 메시지 포맷**: 표준화된 인계 권유 메시지
- **담당자 매핑**: 패키지별 담당자 정보

#### 패키지별 전문 영역

| 패키지 | 전문 영역 | 담당 역할 |
|--------|----------|----------|
| semo-po | 요구사항 기획 & 에픽 관리 | PO |
| semo-qa | 품질 관리 & 테스트 | QA |
| semo-next | Next.js 프론트엔드 개발 | Frontend |
| semo-backend | Spring Boot 백엔드 개발 | Backend |
| semo-infra | DevOps & CI/CD | DevOps |
| semo-pm | 프로젝트 관리 & 스프린트 | PM |
| semo-design | UI/UX 디자인 & 핸드오프 | Designer |
| semo-ms | 마이크로서비스 아키텍처 | MS Dev |

#### 사용 예시

```text
# semo-pm에서 에픽 생성 요청 시
입력: "에픽 만들어줘"

출력:
[SEMO] Cross-Package: 이 요청은 **semo-po**의 전문 영역입니다.

### 권장 조치
| 방법 | 설명 |
|------|------|
| **패키지 전환** | `[po] 에픽 만들어줘` 접두사로 다시 요청 |
| **담당자 문의** | PO 담당 (Roki)에게 문의 |
```
