## ✨ Features

### 용어 변환 가이드 추가 (#94)

QA 테스트 관련 스킬에 **비개발자 친화적 용어 변환 가이드**를 추가했습니다.

#### 적용 스킬

- `execute-test` - 테스트 실행 가이드
- `report-test-result` - 테스트 결과 보고
- `report-bug` - 버그 리포트 생성

#### 주요 용어 변환

| 개발 용어 | 일반 용어 |
|----------|----------|
| CRUD | 작성/조회/수정/삭제 |
| 404 에러 | 페이지를 찾을 수 없음 |
| 500 에러 | 서버 오류 |
| API | 서버 연동 |
| 리다이렉트 | 다른 페이지로 이동 |
| enum 에러 | 선택 값 오류 |
| validation | 입력 값 검증 |
| timeout | 응답 시간 초과 |
| Pass/Fail | 정상/실패 |
| Critical/Major/Minor | 심각/중요/경미 |

## 🎯 기대 효과

- 비개발자(QA, PM, 기획자)도 테스트 결과를 쉽게 이해
- 개발팀과 비개발팀 간 커뮤니케이션 원활화
- 버그 리포트 가독성 향상
