# Browser Test Workflow

> `/browser-test` 명령어로 호출되는 워크플로우

## 트리거

```
/browser-test {url}
```

## 프로세스

### 1. 브라우저 실행

브라우저 서브에이전트를 활용하여 지정된 URL을 엽니다.

### 2. 시각적 검증

다음 항목을 검증합니다:
- 레이아웃 정확성
- 반응형 동작
- 인터랙션 기능
- 로딩 상태
- 에러 상태

### 3. 스크린샷 캡처

검증 결과를 스크린샷으로 기록합니다:
- Desktop (1920x1080)
- Tablet (768x1024)
- Mobile (375x667)

### 4. 결과 보고

검증 결과를 마크다운으로 정리합니다.

---

## 예시

### 입력
```
/browser-test http://localhost:3000/office
```

[SAX] Browser Test: http://localhost:3000/office
