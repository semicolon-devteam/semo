# QuickNote Design Brief

> SEMO Greenfield 워크플로우 Discovery Phase 입력 문서

---

## Problem Statement

사용자가 빠르게 메모를 작성하고 관리할 수 있는 간단한 기능이 필요합니다.

현재 팀원들은 다양한 도구(슬랙, 노션, 메모장 등)를 사용하여 메모를 작성하고 있어 정보가 분산되어 있습니다. 프로젝트 내에서 간단한 메모를 빠르게 작성하고 조회할 수 있는 통합된 기능이 필요합니다.

---

## Goals

### Primary Goals

1. **노트 생성**: 제목과 내용을 입력하여 새 노트를 빠르게 생성
2. **노트 조회**: 저장된 노트 목록을 확인하고 상세 내용 열람
3. **노트 삭제**: 더 이상 필요 없는 노트 제거

### Secondary Goals

- 직관적인 UI로 빠른 접근성 제공
- 모바일 반응형 디자인
- 수정일 기준 최신 노트 우선 표시

### Non-Goals (Out of Scope)

- Rich text 편집 (마크다운, 서식 등)
- 노트 공유/협업 기능
- 태그/카테고리 분류
- 전문 검색 기능
- 노트 내보내기/가져오기

---

## Target Users

### Primary User

**팀 내 개발자/PM**
- 빠른 메모가 필요한 순간에 사용
- 회의 중 간단한 기록
- 아이디어 임시 저장

### User Characteristics

- 기술적 배경 있음
- 빠른 입력/조회 선호
- 복잡한 기능보다 단순함 선호

## User Scenarios

### Scenario 1: 노트 목록 확인

```
1. 사용자가 앱에 접속한다
2. 노트 목록 페이지가 표시된다
3. 최근 수정된 노트가 위에 표시된다
4. 각 노트의 제목과 수정일을 확인할 수 있다
```

### Scenario 2: 새 노트 작성

```
1. 사용자가 "새 노트" 버튼을 클릭한다
2. 노트 작성 폼이 표시된다
3. 제목(필수)을 입력한다
4. 내용(선택)을 입력한다
5. "저장" 버튼을 클릭한다
6. 노트가 저장되고 목록 페이지로 이동한다
```

### Scenario 3: 노트 상세 보기

```
1. 사용자가 노트 목록에서 특정 노트를 클릭한다
2. 노트 상세 페이지가 표시된다
3. 제목, 내용, 생성일, 수정일을 확인할 수 있다
4. "뒤로" 버튼으로 목록으로 돌아갈 수 있다
```

### Scenario 4: 노트 삭제

```
1. 사용자가 노트 상세 페이지에서 "삭제" 버튼을 클릭한다
2. 확인 다이얼로그가 표시된다
3. "확인"을 클릭하면 노트가 삭제된다
4. 노트 목록 페이지로 이동한다
```

## Technical Constraints

### Required Stack

- **Frontend**: Next.js 14+ (App Router)
- **Backend**: Supabase (PostgreSQL + RPC Functions)
- **State**: TanStack Query
- **Styling**: Tailwind CSS

### Architecture

- DDD 4-layer 구조
- `_repositories/`: 데이터 접근 계층
- `_api-clients/`: Supabase 클라이언트
- `_hooks/`: React Query 훅
- `_components/`: UI 컴포넌트

### Validation Rules

| 필드 | 규칙 |
|------|------|
| 제목 | 필수, 1-100자 |
| 내용 | 선택, 최대 5000자 |

---

## Success Metrics

| 지표 | 목표 |
|------|------|
| 노트 생성 시간 | 30초 이내 |
| 페이지 로딩 | 2초 이내 |
| 테스트 커버리지 | 80% 이상 |
| Lighthouse 점수 | 90점 이상 |
