# 3.15.0 — KB/Ontology CLI

## ✨ New Features

### `semo kb` — Knowledge Base 관리
- `semo kb pull` — DB에서 KB를 로컬 `.kb/`로 내려받기
- `semo kb push` — 로컬 `.kb/` 데이터를 DB에 업로드
- `semo kb status` — KB 동기화 상태 확인
- `semo kb list` — KB 항목 목록 조회 (`--domain`, `--format json`)
- `semo kb search <query>` — KB 텍스트 검색 (Phase 2에서 시맨틱 검색 추가 예정)
- `semo kb diff` — 로컬 vs DB 차이 비교
- `semo kb sync` — 양방향 동기화 (pull → 온톨로지 포함)

### `semo onto` — 온톨로지 관리
- `semo onto list` — 정의된 온톨로지 도메인 목록
- `semo onto show <domain>` — 도메인 스키마 상세 조회
- `semo onto validate [domain]` — KB 항목의 스키마 준수 검증 (`--all` 지원)

### DB Schema
- `semo.knowledge_base` — 팀 공통 KB 테이블
- `semo.bot_knowledge` — 봇별 KB 테이블
- `semo.ontology` — 도메인별 온톨로지 정의
- 자동 `updated_at` + `version` 트리거
- 기본 도메인 시드: team, project, decision, process, infra

### Local `.kb/` Structure
- `.kb/team.json` — 공통 KB 캐시
- `.kb/bot.json` — 봇별 KB 캐시
- `.kb/ontology/*.json` — 온톨로지 스키마 캐시
- `.kb/.sync-state.json` — 동기화 메타데이터
