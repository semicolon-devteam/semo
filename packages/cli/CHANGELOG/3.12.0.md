# v3.12.0 - BMad Greenfield Workflow 지원

## 주요 변경사항

### 새로운 기능

1. **BMad Greenfield Workflow 지원**
   - `/SEMO:workflow:greenfield` 커맨드 추가
   - 4-Phase 워크플로우: Discovery → Planning → Solutioning → Implementation
   - DB 기반 워크플로우 진행도 추적

2. **워크플로우 관리 스킬 추가**
   - `workflow-start`: 워크플로우 인스턴스 생성 및 시작
   - `workflow-progress`: 진행 상황 조회
   - `workflow-resume`: 중단된 워크플로우 재개

3. **BMad 스킬 추가**
   - `design-user-flow`: UX 사용자 흐름 설계
   - `design-tests`: 구현 전 테스트 케이스 설계 (TDD)

### DB 스키마 변경

새로운 테이블 추가:
- `workflow_nodes`: 워크플로우 노드 정의
- `workflow_edges`: 노드 간 연결
- `workflow_node_executions`: 노드 실행 기록

기존 테이블 확장:
- `workflow_definitions`: command_name, start_node_id 컬럼 추가
- `workflow_instances`: instance_name, current_node_id 컬럼 추가

### 새로운 에이전트

- Analyst (정분석): Discovery/분석 담당
- UX Designer (유엑스): UX/UI 설계 담당
- SM (김스크럼): 스프린트/스크럼 관리 담당

## 업그레이드 방법

```bash
# 1. CLI 업데이트
npm i -g @team-semicolon/semo-cli@latest

# 2. 기존 프로젝트 업데이트
cd your-project
semo update

# 3. DB 마이그레이션 (Supabase 대시보드에서 실행)
# supabase/migrations/20260120000_bmad_workflow_nodes.sql
# supabase/migrations/20260120001_bmad_greenfield_seed.sql
```

## 워크플로우 사용법

```bash
# 새 프로젝트 시작
/SEMO:workflow:greenfield

# 진행 상황 확인
skill:workflow-progress

# 중단된 워크플로우 재개
skill:workflow-resume
```

## 호환성

- Node.js >= 18.0.0
- Claude Code >= 1.0.0
- Supabase (Cloud 또는 Self-hosted)
