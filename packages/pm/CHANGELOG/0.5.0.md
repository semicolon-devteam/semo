# [0.5.0] - 2025-12-01

## Added

- **start-task Skill**: Task 작업 시작 시 필요한 모든 필드를 한 번에 설정하는 새로운 Skill
  - 상태 → '작업중' 자동 변경
  - 시작일 → 오늘 날짜 자동 설정
  - 이터레이션 → 현재(Current) 이터레이션 자동 할당
  - 단일/복수 Task 일괄 처리 지원
  - 특정 이터레이션 지정 옵션 (`--iteration`)

- **/SAX:sprint start** 커맨드: 작업 시작을 위한 CLI 인터페이스
  - `start #123` - 단일 Task 작업 시작
  - `start #123 #124 #125` - 복수 Task 일괄 시작
  - `start #123 --iteration "12월 1/4"` - 특정 이터레이션 지정

## Changed

- **sprint-master Agent**: start-task Skill 연동 추가
  - 호출 Skills 목록에 start-task 추가
  - Task 작업 시작 워크플로우 추가

- **orchestrator routing-table**: 작업 시작 관련 라우팅 규칙 추가
  - sprint-master 트리거 키워드: "시작", "start", "작업중", "in progress", "착수"
  - Skill 직접 라우팅: start-task 추가

- **/SAX:sprint** 커맨드 문서: start action 추가
  - Sprint 전체 워크플로우에 작업 시작 단계 추가
  - 연관 Skills에 start-task 추가

## Workflow Integration

작업 시작이 Sprint 워크플로우에 자연스럽게 통합됨:

```text
Sprint 계획
    ↓
set-estimate (작업량 설정)
    ↓
assign-to-sprint (Sprint 할당) - 선택적
    ↓
start-task (작업 시작)  ← NEW
├─ 상태 → 작업중
├─ 시작일 → 오늘
└─ 이터레이션 → Current
    ↓
작업 진행...
    ↓
close-sprint (Sprint 종료)
```

## Key Feature

**작업 시작 시 이터레이션 자동 할당**: Sprint에 사전 할당하지 않은 Task도 작업 시작 시 자동으로 현재 이터레이션에 포함됩니다.
