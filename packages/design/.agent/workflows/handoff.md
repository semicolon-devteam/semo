# Design Handoff Workflow

> `/handoff` 명령어로 호출되는 워크플로우

## 트리거

```
/handoff {component_name}
/handoff {figma_url}
```

## 프로세스

### 1. 소스 식별

대상 디자인의 소스를 확인합니다:
- 이전에 생성한 목업
- Figma URL
- 구두 설명

### 2. 스펙 추출

디자인에서 다음 정보를 추출합니다:
- 레이아웃 구조
- 색상 값
- 타이포그래피
- 스페이싱
- 인터랙션 상태

### 3. 문서 생성

표준 핸드오프 문서 형식으로 작성합니다.

### 4. 저장

`design-handoff.md` 또는 지정된 경로에 저장합니다.

## 예시

### 입력
```

```

### 출력
```
[SEMO] Skill: handoff - 핸드오프 문서 생성

## Design Handoff: 로그인 폼

### 1. 개요
- 목적: 사용자 인증
```

## Claude Code 연동

이 문서는 Claude Code와 Antigravity 간의 브릿지 역할을 합니다:

```
[Antigravity]
1. /mockup으로 목업 생성
2. /handoff로 문서 생성
   ↓ (design-handoff.md)
[Claude Code]
3. 문서 참조하여 컴포넌트 구현
4. 코드 리뷰 및 테스트
```

## 체크리스트

핸드오프 문서 완성도 확인:

- [ ] 개요 (목적, 대상)
- [ ] 색상 값 (HEX)
- [ ] 타이포그래피 (폰트, 크기, 굵기)
- [ ] 스페이싱 (패딩, 마진)
- [ ] 상태별 스타일
- [ ] 애니메이션
- [ ] 반응형 (3 breakpoints)
- [ ] 접근성 (ARIA, 키보드)
- [ ] 에셋 링크
