# 개발자 관점 질문 체크리스트

> Epic 작성 시 개발자가 구현 단계에서 할 질문들을 사전에 점검합니다.

## 사용 시점

Epic 생성 **전** 아래 체크리스트를 검토하고, 누락된 항목이 있으면 Epic 본문에 명시합니다.

---

## 1. 데이터 흐름 질문

개발자가 "데이터가 어떻게 흐르지?"라고 물을 수 있는 항목들:

| 질문 | 확인 사항 | Epic에 명시할 내용 |
|------|----------|-------------------|
| 데이터 충돌 시 어떻게 처리? | 동일 데이터를 여러 곳에서 수정할 수 있는가? | 충돌 해결 정책 (최신 우선/병합/경고) |
| A에서 입력 → B에서 수정하면? | 멀티 플랫폼 데이터 동기화 필요? | 동기화 전략 (실시간/폴링/수동) |
| 오프라인에서 입력하면? | 오프라인 지원 필요? | 오프라인 큐잉 정책 |
| 삭제 시 연관 데이터는? | Cascade 삭제? Soft 삭제? | 삭제 정책 명시 |

### 체크리스트

- [ ] 데이터 충돌 해결 정책이 정의되어 있는가?
- [ ] 멀티 플랫폼 간 데이터 흐름이 명확한가?
- [ ] 삭제/수정 시 연관 데이터 처리가 정의되어 있는가?

---

## 2. 시간/계산 로직 질문

개발자가 "이거 어떻게 계산하지?"라고 물을 수 있는 항목들:

| 질문 | 확인 사항 | Epic에 명시할 내용 |
|------|----------|-------------------|
| 일별/월별 집계는? | 집계 주기가 있는가? | 집계 기준 (일/주/월/연) |
| 날짜 기준은? | 생성일? 수정일? 특정 날짜? | 기준 날짜 필드 명시 |
| 일할 계산이 필요한가? | 월 단위 → 일 단위 환산 필요? | 일할 계산 공식 |
| 타임존은? | 사용자별 타임존? 서버 타임존? | 타임존 처리 정책 |

### 체크리스트

- [ ] 집계/계산 로직이 명확하게 정의되어 있는가?
- [ ] 날짜/시간 기준이 명시되어 있는가?
- [ ] 도메인 특화 계산 공식이 필요한 경우 명시되어 있는가?

---

## 3. 플랫폼 제약사항 질문

개발자가 "이거 기술적으로 가능해?"라고 물을 수 있는 항목들:

### PWA 제약

| 기능 | 제약사항 | 대안 |
|------|---------|------|
| 카메라 커스텀 UI | 불가 (브라우저 기본만) | 촬영 후 크롭 UI 제공 |
| 푸시 알림 | iOS Safari 제한적 | 이메일/SMS 병행 |
| 파일 시스템 접근 | 제한적 | File API 활용 |
| 백그라운드 동기화 | 제한적 | 앱 활성화 시 동기화 |

### 웹 제약

| 기능 | 제약사항 | 대안 |
|------|---------|------|
| 로컬 파일 접근 | 사용자 선택 필요 | 드래그앤드롭 UI |
| 클립보드 | HTTPS 필수 | 수동 복사 안내 |

### 모바일 제약

| 기능 | 제약사항 | 대안 |
|------|---------|------|
| 화면 크기 | 320px 최소 | 반응형 필수 |
| 터치 영역 | 44px 최소 | 버튼 크기 준수 |
| 키보드 | 화면 가림 | 스크롤/포커스 처리 |

### 체크리스트

- [ ] 타겟 플랫폼(PWA/웹/네이티브)이 명시되어 있는가?
- [ ] 플랫폼 제약으로 불가능한 기능이 있다면 대안이 제시되어 있는가?
- [ ] 반응형/접근성 요구사항이 정의되어 있는가?

---

## 4. 도메인 지식 질문

개발자가 "이 업계에서는 어떻게 하지?"라고 물을 수 있는 항목들:

### 도메인별 기본 개념

| 도메인 | 확인 필요 개념 |
|--------|---------------|
| 회계/재무 | 일할 계산, 발생주의/현금주의, 결산 |
| 이커머스 | 재고 관리, 주문 상태 머신, 환불 정책 |
| 예약 시스템 | 중복 예약 방지, 취소 정책, 노쇼 처리 |
| 콘텐츠 관리 | 버전 관리, 발행 워크플로우, 권한 |
| 사용자 인증 | 세션 관리, 토큰 갱신, 다중 디바이스 |

### 체크리스트

- [ ] 해당 도메인의 기본 용어/개념이 정의되어 있는가?
- [ ] 업계 표준 프로세스가 반영되어 있는가?
- [ ] 엣지 케이스(예외 상황)가 정의되어 있는가?

---

## 5. 엣지 케이스 질문

개발자가 "이런 경우는?"이라고 물을 수 있는 항목들:

### 공통 엣지 케이스

- [ ] 빈 상태 (데이터 없음) 처리가 정의되어 있는가?
- [ ] 대량 데이터 (1000건 이상) 처리가 고려되어 있는가?
- [ ] 동시 접속/수정 시 처리가 정의되어 있는가?
- [ ] 권한 없는 접근 시 처리가 정의되어 있는가?
- [ ] 네트워크 오류 시 재시도 정책이 있는가?

---

## 사용 예시

### Before (기존 Epic)

```markdown
## 사용자 스토리
- 사용자는 매출을 입력할 수 있다
- 사용자는 브리핑을 볼 수 있다
```

### After (체크리스트 적용 후)

```markdown
## 사용자 스토리
- 사용자는 매출을 입력할 수 있다
  - **데이터 충돌**: 모바일/PC 동시 입력 시 최신 타임스탬프 우선
  - **오프라인**: 로컬 저장 후 온라인 시 자동 동기화

- 사용자는 브리핑을 볼 수 있다
  - **일할 계산**: 월세 30만원 → 일 1만원 (30일 기준)
  - **집계 기준**: 매일 자정(KST) 기준 전일 데이터 집계
```
