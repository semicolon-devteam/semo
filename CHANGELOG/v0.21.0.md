## Added

### list-bugs Skill 추가 (#6)

GitHub Projects '타입' 필드 기반 버그 이슈 조회 스킬:

#### 주요 기능

| 기능 | 설명 |
|------|------|
| **Projects 타입 기준** | 라벨 대신 `이슈관리` Projects의 `타입` 필드로 필터링 |
| **GraphQL API** | GitHub Projects V2 GraphQL API 활용 |
| **일관된 조회** | 레포별 라벨 naming 차이 문제 해결 |

#### 사용 예시

```text
"버그 이슈 조회해줘"
"버그 목록 확인"
"열린 버그 있어?"
```

#### 파일 추가

| 파일 | 설명 |
|------|------|
| `skills/list-bugs/SKILL.md` | 버그 이슈 조회 스킬 |

#### 기존 방식의 문제점

```bash
# 레포마다 라벨 naming이 다름
gh issue list --label "bug"      # 어떤 레포
gh issue list --label "BugFix"   # 다른 레포
gh issue list --label "🐛"       # 또 다른 레포
```

#### 새 방식

```bash
# Projects 타입=버그 필터 조회 (GraphQL)
# 모든 레포에서 일관된 결과
```
