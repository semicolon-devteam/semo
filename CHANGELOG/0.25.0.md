# 0.25.0

> review-task Skill 추가 및 피드백 반영 (Orchestrator, Advisor 개선)

## Added

- **review-task Skill**: GitHub 태스크 이슈카드 기반 구현 완료 리뷰
  - PR 전 태스크카드 종합 검토 자동화
  - 6단계 리뷰 프로세스:
    1. 메타데이터 검증 (Draft 상태, 브랜치명 규칙)
    2. 테스트 구조 검증 (개발자 테스트 vs QA 테스트 분리)
    3. 아키텍처 검증 (DDD 레이어, 팀 컨벤션)
    4. 기능 구현 확인 (Acceptance Criteria 충족)
    5. 품질 게이트 (Lint, TypeScript, Test)
    6. PR 리뷰 코멘트 작성 (APPROVE/COMMENT/REQUEST_CHANGES)
  - `references/review-phases.md` - 리뷰 단계별 상세 로직
  - `references/output-format.md` - 출력 형식 (기본/축약/JSON)

## Features

### 메타데이터 검증
- `[Draft]` 접두사 및 `draft` 라벨 제거 확인
- 브랜치명 규칙 준수 확인 (`{이슈번호}-{영문-기능명}`)

### 테스트 구조 검증
- 개발자 테스트(자동화) / QA 테스트(수동) 섹션 분리 확인
- 관련 테스트 코드 존재 여부 확인

### 아키텍처 검증
- DDD 4레이어 구조 준수 확인
- 팀 Constitution 원칙 준수 (Server-First, Type Safety)

### 기능 구현 확인
- Acceptance Criteria 항목별 코드 매칭
- 구현율 자동 계산

### PR 리뷰 코멘트 작성 (Phase 6)

- 리뷰 완료 후 대상 PR에 자동 코멘트 작성
- 리뷰 타입 자동 결정:
  - `APPROVE`: Critical 0건
  - `COMMENT`: Critical 0건, Warning 1건 이상
  - `REQUEST_CHANGES`: Critical 1건 이상
- `gh pr review` 명령어로 GitHub PR에 직접 리뷰 제출

## Usage

```bash
# 현재 브랜치 이슈 자동 감지
"리뷰해줘"

# 이슈 번호 지정
"cm-land#565 리뷰해줘"
```
