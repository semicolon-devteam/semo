# Platform: MVP

> MVP/프로토타입 프로젝트 코드 구현 가이드

---

## 감지 조건

```bash
# 다른 플랫폼 조건이 모두 false일 때 기본값
```

---

## 핵심 원칙

### 1. 속도 우선

- 완벽한 구조보다 **동작하는 코드** 우선
- 복잡한 추상화 지양
- 빠른 피드백 루프

### 2. 리팩토링 용이성

- 변경하기 쉬운 구조
- 과도한 결합 회피
- 명확한 책임 분리

### 3. 핵심 기능 집중

- 부가 기능 후순위
- 최소 기능 검증
- 사용자 피드백 수집 가능 상태

---

## 권장 스택

| 영역 | 권장 도구 | 이유 |
|------|----------|------|
| Frontend | Next.js / React | 빠른 개발, 생태계 |
| Backend | Supabase / Firebase | BaaS로 백엔드 최소화 |
| DB | Supabase (PostgreSQL) | SQL 기반, 확장 용이 |
| 인증 | Supabase Auth | 즉시 사용 가능 |
| 배포 | Vercel / Netlify | 무료, 자동 배포 |

---

## 구현 가이드라인

### DO (권장)

```
✅ 단순한 구조 유지
✅ 인라인 스타일/Tailwind 활용
✅ 서버리스 함수 활용
✅ 기존 컴포넌트 라이브러리 활용 (shadcn/ui 등)
```

### DON'T (지양)

```
❌ 과도한 폴더 구조
❌ 불필요한 추상화 레이어
❌ 복잡한 상태 관리 (Redux 등)
❌ 마이크로서비스 아키텍처
```
