# 3.7.0

## Changed

- **워크플로우 재설계**: Epic → Task → Speckit 순서로 변경
  - Task Issue가 Source of Truth
  - specs/ 파일은 Task Issue 기반으로 역생성

- **create-issues → create-tasks 리네임 및 재설계**
  - Epic Issue 파싱 후 DDD Layer 기반 Task 분해
  - Layer별 정보 선별 위임 (CONFIG, PROJECT, DATA, TESTS, CODE)
  - Task에 Speckit 체크리스트 포함 (`[ ] specify`, `[ ] plan`, `[ ] implement`)
  - 새 트리거: "태스크 만들어줘", "task 생성", "이슈 만들어줘"

- **generate-spec**: Reverse Mode 추가
  - Task Issue → spec.md, plan.md 역생성
  - Task Issue Speckit Progress 자동 업데이트 (체크 + 링크)
  - Forward Mode (기존) 유지

- **ideate**: 출력 메시지 업데이트
  - 다음 단계로 create-tasks 안내 추가
  - 워크플로우 다이어그램에서 Specification Phase → Task 생성으로 변경

## Added

- `create-tasks/references/layer-delegation.md` - Layer별 정보 위임 상세 가이드
- `generate-spec/references/reverse-mode.md` - Reverse Mode 상세 가이드
