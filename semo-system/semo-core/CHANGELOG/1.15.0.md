# 1.15.0 (2025-12-28)

## Changed

- **routing-base.md 스킬 역할 분리 규칙 추가**
  - tester 스킬 범위 명확화: 테스트 코드 작성/실행만 담당
  - planner 스킬 범위 명확화: 기능 구현/리팩토링 계획만 담당
  - QA 프로세스는 별도 스킬로 분리 (change-to-testing, request-test)
  - Sprint/Roadmap 계획은 별도 Agent로 분리 (sprint-master, roadmap-planner)

- **키워드 우선순위 규칙 추가**
  - QA 프로세스 우선: "테스트 요청해줘" → `request-test` (not `tester`)
  - 구체적 키워드 우선: "테스트 코드 작성" → `tester` (not `implement`)
