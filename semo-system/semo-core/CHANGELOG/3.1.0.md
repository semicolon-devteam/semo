# 3.1.0 (2024-12-30)

## Added

### Agile 역할 기반 에이전트 (9개)

- **po** (Product Owner): Epic 생성, 태스크 관리, 요구사항 명세
- **sm** (Scrum Master): 스프린트 관리, 진행 추적, 회의록
- **architect**: 아키텍처 설계, 도메인 모델링, ADR 작성
- **frontend**: Next.js 개발, UI 구현, 컴포넌트 설계
- **backend**: API 개발, DB 마이그레이션, 서비스 로직
- **dev**: 범용 코드 작성, 버그 수정, 리팩토링
- **qa**: 테스트 설계/실행, 품질 검증, 릴리스 승인
- **devops**: 배포 자동화, 인프라 관리, 롤백 처리
- **teacher**: 기술 교육, 온보딩, 코드 설명

### 에이전트 특징

1. **Internal Routing Map**: 각 에이전트가 상황에 따라 호출할 스킬 매핑
2. **Workflow 정의**: 에이전트별 작업 흐름 문서화
3. **Decision Making**: 상황별 의사결정 기준
4. **Related Agents**: 에이전트 간 연결 관계

## Changed

### Orchestrator 라우팅 테이블 업데이트

- **Skill 직접 라우팅**: 단순 작업은 Skill로 직접 라우팅
- **Agent 라우팅**: 복합 작업은 Agent에게 위임
- **라우팅 결정 기준**: 단순/복합 작업 구분 로직 추가

```text
사용자 요청
    │
    ├─ 단순 작업 → Skill 직접 호출
    │
    └─ 복합 작업 → Agent 호출 → Agent가 스킬 선택
```

## Migration Guide

### 이전 방식 (v3.0.0)
```
Orchestrator → Skill
```

### 새로운 방식 (v3.1.0)
```
Orchestrator → Skill (단순 작업)
Orchestrator → Agent → Skill (복합 작업)
```

### Agent 라우팅 예시

| 요청 | 라우팅 |
|------|--------|
| "커밋해줘" | `skill:git-workflow` (단순) |
| "Epic 만들고 태스크 분배해줘" | `agent:po` (복합) |
| "아키텍처 설계하고 도메인 생성해줘" | `agent:architect` (복합) |
| "테스트하고 배포해줘" | `agent:qa` → `agent:devops` (복합) |
