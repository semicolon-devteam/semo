## 🔄 라우팅 우선순위

키워드 충돌 시 우선순위:

1. **커맨드 패턴** (`/SEMO:*`) → 해당 커맨드
2. **프로젝트 별칭 + 배포** → `deployer`
3. **테스트 관련** → `tester`
4. **코드 관련** → `coder`
5. **기타** → 키워드 매칭

## ⚠️ 엣지 케이스

### 복합 요청

| 테스트 메시지 | 예상 라우팅 | 비고 |
|--------------|------------|------|
| "테스트 작성하고 배포해줘" | ❓ 순차 처리 | tester → deployer |
| "로그인 기능 만들고 테스트해줘" | ❓ 순차 처리 | coder → tester |

### 모호한 요청

| 테스트 메시지 | 예상 라우팅 | 비고 |
|--------------|------------|------|
| "버그" (단독) | ⚠️ 확인 필요 | feedback vs list-bugs |
| "확인해줘" (단독) | ⚠️ 직접 처리 | 라우팅 불가 |

## 검증 방법

```bash
# 설치 환경에서 테스트
claude "로그인 기능 만들어줘"
# 예상: [SEMO] Orchestrator: 의도 분석 완료 → 구현 요청
#       [SEMO] Skill 호출: coder

claude "랜드 stg 배포해줘"
# 예상: [SEMO] Orchestrator: 의도 분석 완료 → 배포 요청
#       [SEMO] Skill 호출: deployer
```
