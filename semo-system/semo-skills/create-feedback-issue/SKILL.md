# create-feedback-issue Skill

> SEMO 패키지에 대한 사용자 피드백을 GitHub 이슈로 생성 (SEMO 공통 Skill)

## Purpose

모든 SEMO 패키지에서 공통으로 사용하는 피드백 수집 Skill입니다.

## Feedback Types

| 유형 | 설명 | 라벨 |
|------|------|------|
| **bug** | 의도한 대로 동작하지 않음 | `bug`, `{package}` |
| **enhancement** | 개선 아이디어, 새 기능 요청 | `enhancement`, `{package}` |

## Workflow

```text
1. 피드백 유형 확인 (버그 or 제안)
   ↓
2. 정보 수집 (질문/결과/기대사항)
   ↓
3. 이슈 생성 (gh CLI)
   ↓
4. 완료 메시지 (이슈 번호 안내)
```

## Execution

### Step 1: 피드백 정보 수집

사용자에게 다음 정보를 수집:
- 피드백 유형 (bug / enhancement)
- 제목
- 상세 내용 (재현 단계, 기대 결과 등)

### Step 2: 이슈 본문 작성

```bash
# 임시 파일에 본문 저장 (긴 텍스트 처리)
cat > /tmp/issue-body.md << 'EOF'
## 피드백 내용

{사용자 입력 내용}

## 재현 단계 (버그인 경우)

1. ...
2. ...

## 기대 결과

...

---
Generated by SEMO Feedback Skill
EOF
```

### Step 3: gh CLI로 이슈 생성

```bash
# 버그 이슈
gh issue create \
  --repo semicolon-devteam/{package} \
  --title "[Feedback] {제목}" \
  --body-file /tmp/issue-body.md \
  --label "bug" \
  --label "{package}"

# 개선 제안
gh issue create \
  --repo semicolon-devteam/{package} \
  --title "[Enhancement] {제목}" \
  --body-file /tmp/issue-body.md \
  --label "enhancement" \
  --label "{package}"
```

### Step 4: 임시 파일 정리

```bash
rm -f /tmp/issue-body.md
```

## Output

```markdown
[SEMO] Feedback: 이슈 생성 완료

✅ 피드백이 등록되었습니다!

**이슈**: semicolon-devteam/{package}#{이슈번호}
**제목**: {이슈 제목}
**유형**: {버그/제안}
```
