## 1. 개요

게임랜드, 플레이랜드, 오피스랜드 3개 서비스에서 **통일된 UI/UX**로 계정 연동 및 포인트 교환 기능을 제공한다.  
사용자 혼란을 최소화하고 일관된 경험을 제공하는 것이 핵심 목표.

**백엔드 API**:
- `ms-authenticator` — 계정 연동 토큰 발급/검증
- `ms-point-exchanger` — 포인트 교환 처리

**프론트엔드 플랫폼**: Next.js (게임랜드, 플레이랜드, 오피스랜드)  
**디바이스 우선순위**: 모바일 우선 (반응형 웹)

## 3. 계정 연동 UI (페이지 A)

### 3.1 기능 개요

- 현재 서비스를 제외한 2개 형제 서비스의 계정 연동 상태 표시
- 미연동 서비스에 대해 연동 토큰 발급 및 입력 플로우 제공
- 이미 연동된 서비스는 해제 불가 (어드민 에스컬레이션)

### 3.2 화면 레이아웃

#### 3.2.1 서비스 카드 목록

```
┌───────────────────────────────────────┐
│  계정 연동 관리                        │
├───────────────────────────────────────┤
│                                       │
│  ┌─────────────────────────────────┐ │
│  │ 🎮 게임랜드                      │ │
│  │ ✅ 연동됨                        │ │
│  │ 연동 일시: 2026-02-15 14:32      │ │
│  │                                  │ │
│  │ [연동 해제 문의하기] ──────────→ │ │
│  └─────────────────────────────────┘ │
│                                       │
│  ┌─────────────────────────────────┐ │
│  │ 🎯 플레이랜드                    │ │
│  │ ❌ 미연동                        │ │
│  │                                  │ │
│  │ [연동 시작하기] ──────────────→  │ │
│  └─────────────────────────────────┘ │
│                                       │
└───────────────────────────────────────┘
```

**카드 구성 요소**:
- 서비스 아이콘 + 이름
- 연동 상태 (✅ 연동됨 / ❌ 미연동) — **시각적 구분 필요** (색상, 배지 등)
- 연동됨: 연동 일시 표시 + "연동 해제 문의하기" 버튼
- 미연동: "연동 시작하기" 버튼

#### 3.2.2 연동 플로우 — 토큰 발급

사용자가 "연동 시작하기" 클릭 시:

```
┌───────────────────────────────────────┐
│  🎯 플레이랜드 계정 연동               │
├───────────────────────────────────────┤
│                                       │
│  1단계: 연동 토큰 발급                │
│                                       │
│  아래 토큰을 복사한 후,               │
│  플레이랜드에서 "계정 연동" 메뉴로    │
│  이동하여 붙여넣기 해주세요.          │
│                                       │
│  ┌─────────────────────────────────┐ │
│  │ abc123-def456-ghi789-jkl012     │ │
│  │ [📋 클립보드 복사] ─────────────→│ │
│  └─────────────────────────────────┘ │
│                                       │
│  ⏱️ 유효시간: 14분 32초 남음          │
│                                       │
│  [닫기]                              │
└───────────────────────────────────────┘
```

**기능 요소**:
- 15분 만료 타이머 (실시간 카운트다운)
- 토큰 텍스트 필드 (읽기 전용)
- "📋 클립보드 복사" 버튼 — 원클릭 복사
- 안내 문구 (타서비스로 이동 유도)

#### 3.2.3 연동 플로우 — 토큰 입력

다른 서비스에서 "계정 연동" 페이지 진입 시:

```
┌───────────────────────────────────────┐
│  계정 연동하기                         │
├───────────────────────────────────────┤
│                                       │
│  다른 랜드 서비스에서 발급받은 연동   │
│  토큰을 입력해주세요.                 │
│                                       │
│  ┌─────────────────────────────────┐ │
│  │ [연동 토큰 입력]                 │ │
│  └─────────────────────────────────┘ │
│                                       │
│  [연동하기]                          │
└───────────────────────────────────────┘
```

**처리 플로우**:
1. 사용자 토큰 입력
2. "연동하기" 클릭
3. globalLoader 표시 (서버 검증 중)
4. 성공 시: 토스트 "✅ 계정이 연동되었습니다" + 카드 상태 업데이트
5. 실패 시: 토스트 "❌ 토큰이 유효하지 않습니다"

#### 3.2.4 연동 해제 문의

"연동 해제 문의하기" 클릭 시:
- 어드민 채팅창 열기 (기존 고객지원 채널 활용)
- 또는 모달: "연동 해제는 관리자에게 문의해주세요" + 채팅 버튼

### 3.3 에러 케이스

| 상황 | UI 응답 |
|------|---------|
| 토큰 만료 | 토스트: "토큰이 유효하지 않습니다" |
| 토큰 형식 오류 | 토스트: "토큰이 유효하지 않습니다" |
| 이미 연동됨 | 토스트: "이미 연동된 계정입니다" |
| 네트워크 오류 | 토스트: "네트워크 오류가 발생했습니다" + 재시도 버튼 |

## 4. 포인트 교환 UI (페이지 B)

### 4.1 기능 개요

- 연동된 서비스 간 포인트 교환
- 출발 서비스는 현재 서비스로 고정
- 도착 서비스는 카드 형식 라디오버튼으로 선택
- 실시간 보유 포인트 표시 + 환율 적용 미리보기

### 4.2 화면 레이아웃

```
┌───────────────────────────────────────┐
│  포인트 교환                           │
├───────────────────────────────────────┤
│                                       │
│  보유 포인트: 12,500P                 │
│                                       │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                       │
│  출발 서비스:                         │
│  🎮 게임랜드 (현재 서비스)            │
│                                       │
│  도착 서비스:                         │
│  ┌─────────────────────────────────┐ │
│  │ ○ 🎯 플레이랜드                  │ │
│  │    연동됨 | 보유: 3,200P         │ │
│  └─────────────────────────────────┘ │
│  ┌─────────────────────────────────┐ │
│  │ ○ 🏢 오피스랜드                  │ │
│  │    미연동 → [계정 연동하기]      │ │
│  └─────────────────────────────────┘ │
│                                       │
│  교환 포인트:                         │
│  ┌─────────────────────────────────┐ │
│  │ [5000]           [전액]          │ │
│  └─────────────────────────────────┘ │
│                                       │
│  환율: 1:1                            │
│  수수료: 0P                           │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  수령 예정: 5,000P                    │
│                                       │
│  [교환하기] ─────────────────────────→│
└───────────────────────────────────────┘
```

**구성 요소**:
1. **보유 포인트**: 현재 서비스의 포인트 잔액 (실시간)
2. **출발 서비스**: 현재 서비스명 표시 (변경 불가)
3. **도착 서비스**: 
   - 카드형 라디오버튼 (2개)
   - 연동됨: 서비스명 + 보유 포인트 표시
   - 미연동: "계정 연동하기" 링크 (계정 연동 페이지로 이동)
4. **교환 포인트 입력**:
   - 숫자 입력 필드
   - "전액" 버튼 (보유 포인트 전체 자동 입력)
   - 입력 시 실시간 유효성 검사 (보유 포인트 초과 불가)
5. **환율/수수료/수령 예정**:
   - 환율: 1:1 (기본값, 향후 변경 가능)
   - 수수료: 0P
   - 수령 예정: 교환 포인트 × 환율 - 수수료 (실시간 계산)
6. **교환하기** 버튼

### 4.3 교환 확정 플로우

사용자가 "교환하기" 클릭 시:

#### 4.3.1 최종 확인 모달

```
┌───────────────────────────────────────┐
│  교환 확인                             │
├───────────────────────────────────────┤
│                                       │
│  아래 내용으로 교환하시겠습니까?       │
│                                       │
│  출발: 🎮 게임랜드                    │
│  도착: 🎯 플레이랜드                  │
│  교환 포인트: 5,000P                  │
│  수령 포인트: 5,000P                  │
│                                       │
│  [취소]  [확인] ─────────────────────→│
└───────────────────────────────────────┘
```

#### 4.3.2 처리 중 (globalLoader)

- 모달 닫힘 + globalLoader 표시
- 서버 응답 대기 (3초 이내)

#### 4.3.3 완료 (토스트 + refetch)

- 성공: 토스트 "✅ 교환이 완료되었습니다"
- 보유 포인트 자동 갱신 (refetch)
- 입력 필드 초기화

### 4.4 에러 케이스

| 상황 | UI 응답 |
|------|---------|
| 연동 안 된 서비스 선택 | "먼저 계정을 연동하세요" 안내 (입력 단계에서 차단) |
| 잔액 부족 | 입력 필드에 실시간 검증 (빨간색 테두리 + 에러 메시지) |
| 0 포인트 입력 | "1 이상의 포인트를 입력하세요" |
| 변환 후 0P | 토스트: "변환 후 금액이 0입니다" |
| 처리 중 중복 요청 | 토스트: "처리 중입니다" |
| 네트워크 오류 | 토스트: "네트워크 오류가 발생했습니다" + 재시도 버튼 |

## 6. API 연동 스펙

### 6.1 계정 연동 API (ms-authenticator)

#### 6.1.1 토큰 발급

**Endpoint**: `POST /api/link/request`

**Request**:
```json
{
  "requester_identity_id": 123,
  "target_service_id": "play-land"
}
```

**Response**:
```json
{
  "verification_token": "abc123-def456-ghi789-jkl012",
  "expires_at": "2026-03-01T08:15:00Z"
}
```

#### 6.1.2 토큰 검증 (연동 확정)

**Endpoint**: `POST /api/link/verify`

**Request**:
```json
{
  "verification_token": "abc123-def456-ghi789-jkl012",
  "target_service_id": "play-land",
  "target_auth_user_id": "uuid-of-play-land-user"
}
```

**Response**:
```json
{
  "unified_user_id": "unified-uuid",
  "verified_at": "2026-03-01T07:32:00Z"
}
```

### 6.2 포인트 교환 API (ms-point-exchanger)

#### 6.2.1 교환 요청

**Endpoint**: `POST /api/exchange/request`

**Headers**:
```
X-API-Key: <service-api-key>
```

**Request**:
```json
{
  "idempotency_key": "client-generated-uuid",
  "unified_user_id": "unified-uuid",
  "source_service_id": "game-land",
  "source_point_code": "GP",
  "source_auth_user_id": "source-user-uuid",
  "source_amount": 5000,
  "target_service_id": "play-land",
  "target_point_code": "PP",
  "target_auth_user_id": "target-user-uuid"
}
```

**Response** (성공):
```json
{
  "id": "exchange-request-uuid",
  "status": "SUCCESS",
  "source_amount": 5000,
  "target_amount": 5000,
  "exchange_rate": 1.0,
  "completed_at": "2026-03-01T07:35:00Z"
}
```

**Response** (실패):
```json
{
  "error": "USER_NOT_LINKED",
  "message": "유저가 대상 서비스에 연동되지 않았습니다"
}
```

#### 6.2.2 교환 내역 조회

**Endpoint**: `GET /api/exchange/history?unified_user_id=<uuid>&page=1&limit=10`

**Response**:
```json
{
  "total": 42,
  "page": 1,
  "limit": 10,
  "items": [
    {
      "id": "exchange-request-uuid",
      "source_service_id": "game-land",
      "target_service_id": "play-land",
      "source_amount": 5000,
      "target_amount": 5000,
      "status": "SUCCESS",
      "created_at": "2026-02-28T14:32:00Z"
    }
  ]
}
```

```typescript
// validators/pointExchange.ts
export const validateExchangeAmount = (amount: number, balance: number): ValidationResult => {
  if (amount <= 0) return { valid: false, message: "1 이상의 포인트를 입력하세요" };
  if (amount > balance) return { valid: false, message: "보유 포인트를 초과했습니다" };
  return { valid: true };
};
```

### 7.5 에러 처리

```typescript
// utils/errorHandler.ts
export const handleExchangeError = (error: ApiError): ToastMessage => {
  switch (error.code) {
    case "USER_NOT_LINKED":
      return { type: "error", message: "먼저 계정을 연동하세요" };
    case "INVALID_AMOUNT":
      return { type: "error", message: "유효하지 않은 금액입니다" };
    case "EXCHANGE_IN_PROGRESS":
      return { type: "warning", message: "처리 중입니다" };
    default:
      return { type: "error", message: "오류가 발생했습니다" };
  }
};
```

## 8. 디자인 요청 사항 (DesignClaw)

### 8.1 필수 화면

1. **계정 연동 페이지**
   - 서비스 카드 목록 (연동/미연동 상태 시각적 구분)
   - 토큰 발급 모달 (토큰 텍스트 + 복사 버튼 + 타이머)
   - 토큰 입력 폼

2. **포인트 교환 페이지**
   - 교환 폼 (출발/도착 서비스 + 금액 입력 + 미리보기)
   - 최종 확인 모달

3. **교환 내역 페이지**
   - 내역 리스트 (카드형 또는 테이블)
   - 페이지네이션

### 8.2 디자인 키워드

- **모바일 우선** (반응형 웹)
- **일관성**: 3개 서비스에서 동일한 UI/UX
- **명확성**: 연동 상태, 처리 상태를 한눈에 파악 가능
- **친절함**: 에러 메시지, 안내 문구가 자연스럽고 명확
- **신뢰감**: 포인트 교환은 금전 관련이므로 확정 전 확인 단계 필수

### 8.3 참고 스타일

- 게임랜드 기존 UI 톤앤매너 유지
- 카드형 레이아웃 선호 (모바일 친화적)
- 토스트는 기존 자체 구현 Toast 스타일 활용
