# Epic: 기업회원 오피스캐시 결제 및 홍보 시스템

## 📋 배경 및 목적

### 배경
- office-land는 현재 개인회원 대상 포인트 구매 시스템을 구현 중
- 기업회원에게는 별도의 마케팅 수단(오피스 홍보)을 제공할 필요
- 랜딩페이지에 오피스 정보를 노출하여 기업회원의 인지도 향상 및 수익 창출

### 목적
- 기업회원 전용 포인트 타입(`OFFICE_CASH`) 도입
- 오피스캐시를 활용한 랜딩페이지 홍보 시스템 구축
- 자동 연장 및 알림 기능으로 사용자 편의성 제고

---

## 🎯 핵심 요구사항

### 1. 오피스캐시(OFFICE_CASH) 포인트 타입 추가
- **비율**: 원화(KRW) 1:1 충전
- **사용 대상**: 기업회원만 구매 및 사용 가능
- **용도**: 랜딩페이지 오피스 정보 홍보

### 2. 오피스캐시 구매 기능
- **진입점**: 마이페이지 메뉴
- **구매 프로세스**:
  1. 기업회원 로그인
  2. 마이페이지 → "오피스캐시 충전" 메뉴
  3. 충전 금액 선택 (또는 직접 입력)
  4. 결제 수단 선택 (기존 결제 시스템 연동)
  5. 결제 완료 후 오피스캐시 즉시 충전

### 3. 오피스 홍보 기능
- **홍보 영역**: 랜딩페이지 상단 또는 주요 섹션
- **노출 기간**: 캐시 소모 시점부터 **30일간**
- **홍보 프로세스**:
  1. 기업회원이 본인 오피스 선택
  2. 오피스캐시로 홍보 비용 결제
  3. 랜딩페이지에 오피스 정보 노출 시작
  4. 30일 후 자동 만료

### 4. 홍보 기간 관리
- **만료 3일 전 알림**:
  - 알림 채널: 이메일 / 앱 푸시 (기존 알림 시스템 연동)
  - 알림 내용: "오피스 홍보 기간이 3일 후 만료됩니다. 연장하시겠습니까?"
  - 연장 링크 제공

- **자동 연장 처리**:
  - 만료 시점에 플랫폼이 자동으로 연장 처리
  - 연장 비용: 동일 금액의 오피스캐시 차감
  - 조건: 기업회원 계정에 충분한 오피스캐시 잔액 보유
  - 잔액 부족 시: 홍보 종료 + 알림 발송

## 👥 사용자 시나리오

### 시나리오 1: 오피스캐시 충전 및 홍보 등록
1. 기업회원 A가 로그인
2. 마이페이지 → "오피스캐시 충전" 클릭
3. 10만원 충전 (오피스캐시 100,000 획득)
4. "내 오피스 홍보하기" 메뉴 진입
5. 홍보할 오피스 선택
6. 홍보 비용 5만 오피스캐시 결제
7. 랜딩페이지에 오피스 정보 노출 시작 (30일간)

### 시나리오 2: 만료 3일 전 알림 수신
1. 홍보 등록 27일 후, 기업회원 A에게 알림 발송
2. "홍보 기간이 3일 후 만료됩니다" 메시지 표시
3. 연장 링크 클릭 → 오피스캐시 5만 차감 → 추가 30일 연장

### 시나리오 3: 자동 연장 처리 (잔액 충분)
1. 홍보 기간 30일 만료 시점 도달
2. 시스템이 기업회원 A의 오피스캐시 잔액 확인 (50,000 보유)
3. 자동으로 오피스캐시 50,000 차감
4. 홍보 기간 추가 30일 연장
5. 기업회원에게 "자동 연장되었습니다" 알림 발송

### 시나리오 4: 자동 연장 실패 (잔액 부족)
1. 홍보 기간 30일 만료 시점 도달
2. 시스템이 기업회원 B의 오피스캐시 잔액 확인 (20,000 보유)
3. 잔액 부족으로 자동 연장 실패
4. 홍보 종료 처리
5. 기업회원에게 "잔액 부족으로 홍보가 종료되었습니다" 알림 발송

## 🧪 테스트 케이스

### TC-1: 오피스캐시 충전
- **Given**: 기업회원이 마이페이지에 접속
- **When**: 10만원 충전 요청
- **Then**: 오피스캐시 100,000 증가

### TC-2: 오피스 홍보 등록
- **Given**: 오피스캐시 50,000 보유
- **When**: 홍보 비용 50,000 결제
- **Then**: 랜딩페이지에 오피스 정보 노출, 30일 타이머 시작

### TC-3: 만료 3일 전 알림
- **Given**: 홍보 등록 27일 경과
- **When**: 시스템 스케줄러 실행
- **Then**: 기업회원에게 만료 알림 발송

### TC-4: 자동 연장 성공
- **Given**: 홍보 기간 30일 만료, 오피스캐시 50,000 보유
- **When**: 만료 시점 도달
- **Then**: 오피스캐시 50,000 차감, 홍보 기간 30일 연장

### TC-5: 자동 연장 실패
- **Given**: 홍보 기간 30일 만료, 오피스캐시 20,000 보유
- **When**: 만료 시점 도달
- **Then**: 홍보 종료, 잔액 부족 알림 발송

### TC-6: 홍보 중복 방지
- **Given**: 이미 홍보 중인 오피스
- **When**: 동일 오피스 추가 홍보 시도
- **Then**: "이미 홍보 중입니다" 에러 메시지 표시

## 🛠️ 기술 스택 제안

### Backend
- **포인트 타입 확장**: 기존 포인트 시스템에 `OFFICE_CASH` enum 추가
- **트랜잭션 관리**: 충전/소모 시 원자성 보장 (DB 트랜잭션)
- **스케줄러**: 만료 알림 및 자동 연장 처리 (Cron Job 또는 스케줄 워커)

### Frontend
- **마이페이지 UI**: 오피스캐시 충전 페이지
- **홍보 관리 페이지**: 홍보 중인 오피스 목록, 잔여 기간 표시
- **랜딩페이지**: 홍보 오피스 노출 영역 (카드 또는 배너 형태)

### Database
- **테이블 추가/수정**:
  - `point_transactions`: 오피스캐시 충전/소모 기록
  - `office_promotions`: 홍보 정보 (오피스 ID, 시작일, 종료일, 자동연장 여부)

### 알림
- **채널**: 이메일, 앱 푸시 (기존 알림 시스템 재사용)
- **트리거**: 만료 3일 전, 자동 연장 성공/실패
