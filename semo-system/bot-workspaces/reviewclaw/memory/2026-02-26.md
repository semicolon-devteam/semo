## 🚨 PR #169 Regression 재분석 (18:31, SemiClaw 긴급 요청)

**제보자**: Harry Lee
**문제**: PR #169가 SSR → CSR regression 발생

### Critical Issue
**커밋 f8a65b0** (`src/app/page.tsx`):
- **Before**: `getDebates()` + `getCloseDebates()` + `fetchLatestNews()` (3개 서버 fetch)
- **After**: `closeDebatesOptions()` + `latestNewsOptions()` (2개만 prefetch)
- **→ 메인 토론 목록 `getDebates()` prefetch 누락**

### 결과
- 메인 페이지 토론 목록이 SSR에서 로드되지 않음
- CSR만 로드 → 로딩 스피너 노출 → UX 저하

### 내 리뷰에서 놓친 것
**지적한 것**:
- 🟡 Should Fix: "불필요한 prefetch 중복" (closeDebates, latestNews가 여러 페이지에 중복)

**놓친 것**:
- 🔴 **Critical: "누락된 필수 prefetch"** — getDebates() (메인 토론 목록)

### 근본 원인 분석
1. **Before 코드를 명시적으로 확인하지 않음**
   - diff만 보고 "추가된 것"만 체크
   - "제거된 것"은 체크 안 함

2. **SSR 전환 PR 특성 간과**
   - SSR 전환 = 기능 동등성 유지가 핵심
   - "무엇이 추가됐는가"보다 "무엇이 제거됐는가"가 더 중요

3. **Regression 탐지 로직 없음**
   - "불필요한 중복"을 찾는 데 집중
   - "필수 기능 누락"을 찾는 로직 없음

### 재발 방지 조치
1. ✅ `memory/review-standards.md`에 "SSR 전환 PR 리뷰 체크리스트" 추가
   - Before/After 데이터 fetch 1:1 매핑 확인 필수
   - 제거된 fetch 함수 명시적 탐지
   - SSR 동작 검증 (로딩 스피너 없어야 함)

2. ✅ 위반 사례 (교훈) 문서화
   - PR #169 regression 상세 기록
   - "추가된 것"만 체크하지 말고 **"제거된 것"도 명시적으로 체크**

3. ⏳ 향후 SSR 전환 PR 리뷰 시 새 체크리스트 필수 적용

### 교훈
**"Regression은 추가보다 제거에서 온다"**
- 새로운 코드 추가 = 눈에 잘 띔
- 기존 코드 제거 = 의도적으로 찾지 않으면 놓침
- SSR 전환 PR = 기능 동등성이 핵심, Before/After 명시적 비교 필수
